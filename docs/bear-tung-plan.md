## 3. Tech Stack & Libraries

| Area | Choice | Why |
|---|---|---|
| Framework | **Next.js (App Router) + TypeScript** | industry standard, great learning value, easy Vercel deploy |
| Styling | **Tailwind CSS** | fast, aligns with design tokens |
| Charts | **Recharts** | easy donut/bar/line |
| Excel export | **SheetJS (xlsx)** or **ExcelJS** | build .xlsx on the client |
| Dates | **dayjs** (or native `Date`) | handle months/projection range |
| State | **React Context + hooks** (or **Zustand** if it grows) | hold profile + sync local storage |
| Test | **Vitest** + **@testing-library/react** | unit-test finance logic + components |
| Format/Lint | ESLint + Prettier | code quality |

> No backend / database — all data in local storage, making deployment trivially static/serverless.

## 5. Build Phases / Milestones

### Phase 0 — Setup & Deploy skeleton (half day)
- `npx create-next-app@latest` (TypeScript, Tailwind, ESLint, App Router, src/)
- Install libs: recharts, xlsx, dayjs
- Add Tailwind theme tokens (colors/fonts) from design.md
- Write `README.md`, `CLAUDE.md`, `.gitignore`
- Push to GitHub → connect Vercel → **get a working URL from day one** (deploy often)

### Phase 1 — Data model + Storage + Cash Flow form
- `model/` (types) and `storage/` (local storage + carry-forward)
- Cash flow page: categorized form + line item (set value-change date / debt end date)

### Phase 2 — Finance core (domain layer, OOP + SOLID) ⭐ most important
- Build classes/interfaces per `bear-tung-architecture.md`: model, ratios (Strategy), HealthScoreService, mortgage (LtvPolicy + Factory + Amortization), CoBorrowerService, ProjectionService, repository, config
- Write **full unit tests** with DIP/mocking (edge cases: zero income, zero interest, debt ending mid-way, age-capped term)

### Phase 3 — Health Dashboard
- score gauge + traffic light, ratio cards (with jargon explanations)
- expense donut + comparison bar

### Phase 4 — 5-year Projection
- month selector/slider, score trend line, dashboard updates by selected month

### Phase 5 — Mortgage + Co-borrower
- form (price/age/assumptions/month), result card
- co-borrower checkbox + required co-borrower income

### Phase 6 — Excel Export
- export button → 4 sheets (cash flow, health, mortgage, projection)

### Phase 7 — Polish & Accessibility
- responsive (iPad-first → mobile/desktop), jargon tooltips, reduced-motion, contrast check
- end-to-end QA with real numbers + final deploy

## 6. Deploy on Vercel (very easy for Next.js)

1. **Prepare repo:** push code to GitHub (e.g. repo `bear-tung`).
2. **Sign in to Vercel** with GitHub → **Add New → Project**.
3. **Import** the `bear-tung` repo — Vercel auto-detects Next.js and sets the build command (`next build`).
4. **Environment variables:** none needed (client-only, no secrets).
5. Click **Deploy** → get a URL like `https://bear-tung.vercel.app` in minutes.
6. **Auto deploy:** every push to `main` triggers a build + deploy; PRs/branches get separate **Preview URLs**.
7. (optional) attach a **custom domain** later.

> Tip: deploy from Phase 0 and push often to see real progress early and catch deploy issues sooner.

## 7. Testing Strategy

- **Unit tests (Vitest):** focus on `domain/` — every finance function must be correct.
  - ratios, scoring, amortization, maxLoan (LTV/DSR), term cap, co-borrower, projection carry-forward
- **Component tests (testing-library):** input forms, score/traffic-light rendering
- **Manual QA:** verify against the sample numbers in the spec
- Add CI (GitHub Actions) to run tests before deploy in the future

## 8. Risks & cautions

- **LTV policy change:** the 100% relaxation ends 30 Jun 2027 (extended 1 year by BOT ฉบับที่ 19/2569; was 30 Jun 2026) → implement LTV as date-based config (already in spec); review periodically.
- **Calculation correctness:** must have full unit tests before wiring UI.
- **local storage:** data is tied to the browser/device — tell users that switching devices loses data (DB support later).
- **Money numbers:** watch rounding — store as plain numbers, format only at display.
